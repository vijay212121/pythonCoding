-> Keyword-Driven Testing where reusable actions are stored as methods.

Modular & POM-based design for scalability and maintainability.

-> We follow the Page Object Model (POM) design pattern, where each web page is represented by a separate class. This keeps our locators and 	  interaction methods isolated from the test logic, making the framework easy to update and reusable.

-> The test flow and assertions are managed using TestNG. It helps with:
	1) Parallel execution
	2) Grouping test cases
	3) Parameterization
	4) Generating test reports

-> For reporting, we use Extent Reports to create rich HTML reports. These include:
	1) Test case status (pass/fail/skip)
	2) Screenshots on failure
	3) Logs and system information
