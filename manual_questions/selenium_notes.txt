ðŸ”¹What is Shadow DOM?	
âœ… Shadow DOM is a DOM within a DOM, used to encapsulate the structure, style, and behavior of web components. It hides internals from the main document.

ðŸ”¹Why canâ€™t Selenium interact with Shadow DOM directly?	
âœ…Because shadow elements are outside the standard DOM tree and are encapsulated. Selenium doesn't support direct access.

ðŸ”¹How do you access shadow DOM in Selenium?	
âœ…By using JavaScriptExecutor to fetch the shadow root and then locate elements within it.

ðŸ”¹Can you use XPath for Shadow DOM elements?/Why canâ€™t we use normal locators for shadow elements?	
âœ…No, XPath and regular locators donâ€™t work. You must use JS to pierce the shadow DOM/Because they are encapsulated and don't show up in the regular DOM â€” Seleniumâ€™s findElement canâ€™t access them directly.

ðŸ”¹Whatâ€™s the difference between open and closed shadow DOM?	
âœ…Open shadow DOM can be accessed via JavaScript. Closed shadow DOM cannot be accessed at all â€” it's completely encapsulated.

ðŸ”¹How to check if an element is inside a Shadow DOM?	
âœ…Use browser dev tools. If you see #shadow-root in the structure, itâ€™s inside shadow DOM.

Can Shadow DOMs be nested?	Yes, one shadow host can contain another shadow host. Each level must be accessed step by step.
How to handle nested shadow DOMs in Selenium?	Use JavaScriptExecutor recursively: access outer shadow root â†’ find inner shadow host â†’ access its shadow root â†’ then interact with elements.

Example:
<custom-date-picker>
  #shadow-root
    <input type="date">
</custom-date-picker>

JavascriptExecutor js = (JavascriptExecutor) driver;

// Step 1: Get shadow root element
WebElement shadowHost = driver.findElement(By.cssSelector("custom-date-picker"));
WebElement shadowRoot = (WebElement) js.executeScript("return arguments[0].shadowRoot", shadowHost);

// Step 2: Find element inside shadow root
WebElement dateInput = shadowRoot.findElement(By.cssSelector("input[type='date']"));
dateInput.sendKeys("2025-04-10");

ðŸ”¹ What is a TestNG Listener?
ANS: 
A Listener in TestNG is an interface that listens to events during the test execution lifecycle and allows us to perform actions based on those 
events. For example, we can log test start/end, capture screenshots on failure, or even skip tests programmatically.

ðŸ”¹ Name some commonly used TestNG Listeners

onTestSuccess()
onTestFailure()
onTestSkipped()
onFinish()

ðŸ”¹ How do you implement a listener in TestNG?

-> Create a class that implements ITestListener
-> Override the methods you need
-> Register the listener by adding listener tags in testNG.xml file

ðŸ”¹ How do you use Listeners with Extent Reports?
ANS:
To integrate Extent Reports with TestNG, I implement the ITestListener interface. Inside the listener methods like onTestSuccess(), onTestFailure(), 
and onTestSkipped(), I log the test results using the ExtentTest object provided by Extent Reports.
In the onStart() method, I configure the report using ExtentSparkReporter (or ExtentHtmlReporter in older versions), and attach it to the main ExtentReports object.
Finally, I call extent.flush() in the onFinish() method to generate the final HTML report.
