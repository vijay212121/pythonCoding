Shadow DOM:-  

-> only CssSelector locator can be handled shadow DOM elements while locating Host

🔹What is Shadow DOM?	
✅ Shadow DOM is a DOM within a DOM, used to encapsulate the structure, style, and behavior of web components. It hides internals from the main document.

🔹Why can’t Selenium interact with Shadow DOM directly?	
✅Because shadow elements are outside the standard DOM tree and are encapsulated. Selenium doesn't support direct access.

🔹How do you access shadow DOM in Selenium?	
✅By using JavaScriptExecutor to fetch the shadow root and then locate elements within it.

🔹Can you use XPath for Shadow DOM elements?/Why can’t we use normal locators for shadow elements?	
✅No, XPath and regular locators don’t work. You must use JS to pierce the shadow DOM/Because they are encapsulated and don't show up in the regular DOM — Selenium’s findElement can’t access them directly.

🔹What’s the difference between open and closed shadow DOM?	
✅Open shadow DOM can be accessed via JavaScript. Closed shadow DOM cannot be accessed at all — it's completely encapsulated.

🔹How to check if an element is inside a Shadow DOM?	
✅Use browser dev tools. If you see #shadow-root in the structure, it’s inside shadow DOM.

Can Shadow DOMs be nested?	Yes, one shadow host can contain another shadow host. Each level must be accessed step by step.
How to handle nested shadow DOMs in Selenium?	Use JavaScriptExecutor recursively: access outer shadow root → find inner shadow host → access its shadow root → then interact with elements.

Example:
<custom-date-picker>
  #shadow-root
    <input type="date">
</custom-date-picker>

JavascriptExecutor js = (JavascriptExecutor) driver;

// Step 1: Get shadow root element
WebElement shadowHost = driver.findElement(By.cssSelector("custom-date-picker"));
WebElement shadowRoot = (WebElement) js.executeScript("return arguments[0].shadowRoot", shadowHost);

// Step 2: Find element inside shadow root
WebElement dateInput = shadowRoot.findElement(By.cssSelector("input[type='date']"));
dateInput.sendKeys("2025-04-10");

What are different types of waits in Selenium?
Selenium provides three types of waits to handle synchronization issues:

• Implicit Wait: Sets a default waiting time for all elements throughout the session.
		driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

• Explicit Wait: Waits for specific conditions on particular elements before
performing actions.
		WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
		wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("submitB
		tn")));

• Fluent Wait: Provides more control with custom polling frequency and exception
handling.
		Wait<WebDriver> wait = new FluentWait<>(driver)
		 .withTimeout(Duration.ofSeconds(10))
		 .pollingEvery(Duration.ofMillis(500))
		 .ignoring(NoSuchElementException.class);
		wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("element
		")));

Q. What are Desired Capabilities?
Ans:
Desired capabilities are a set of key-value pairs used in Selenium to set up the environment and settings for a test session. They tell the framework
which browser, platform, device to use.

Example:
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.remote.DesiredCapabilities;

public class Test {
    public static void main(String[] args) {
        DesiredCapabilities caps = new DesiredCapabilities();
        caps.setBrowserName("chrome");
        caps.setVersion("114");
        caps.setCapability("platform", "WINDOWS");

        WebDriver driver = new ChromeDriver(caps);
        driver.get("https://www.example.com");
    }
}
